## 1. Arrays

### Array Methods: `map`, `filter`, `reduce`, `find`, `every`, `some`, etc.

#### `map()`

Creates a **new array** by applying a callback to each element.  
Non-mutating.

```js
const nums = [1, 2, 3];
const doubled = nums.map(n => n * 2);
console.log(doubled); // [2, 4, 6]
```

#### `filter()`

Creates a **new array** containing elements that pass the given condition.  
Non-mutating.

```js
const nums = [1, 2, 3, 4];
const evens = nums.filter(n => n % 2 === 0);
console.log(evens); // [2, 4]
```

#### `reduce()`

Reduces an array to a single value by accumulating results.  
Non-mutating.

```js
const nums = [1, 2, 3];
const sum = nums.reduce((acc, val) => acc + val, 0);
console.log(sum); // 6
```

#### `find()`

Returns the **first element** that satisfies the condition, or `undefined` if none match.

```js
const arr = [10, 20, 30];
const found = arr.find(n => n > 15);
console.log(found); // 20
```

#### `every()` and `some()`

- `every()` → checks if **all** elements pass the condition.
    
- `some()` → checks if **at least one** element passes.
    

```js
const arr = [1, 2, 3, 4];
console.log(arr.every(n => n > 0)); // true
console.log(arr.some(n => n > 3));  // true
```

Other useful methods:

- `forEach()` – iterates without returning a new array.
    
- `findIndex()` – returns index of first match.
    
- `flat()` – flattens nested arrays.
    
- `includes()` – checks for presence of a value.
    
- `sort()` – sorts elements (mutates array).

### Mutating vs Non-Mutating Methods

#### Mutating Methods (change the original array)

`push`, `pop`, `shift`, `unshift`, `splice`, `sort`, `reverse`, `fill`, `copyWithin`

```js
const arr = [1, 2, 3];
arr.push(4); // [1, 2, 3, 4]
arr.reverse(); // [4, 3, 2, 1]
```

#### Non-Mutating Methods (return a new array)

`map`, `filter`, `slice`, `concat`, `reduce`, `flat`, `toSorted`, `toReversed`

```js
const arr = [1, 2, 3];
const newArr = arr.slice(0, 2);
console.log(arr); // [1, 2, 3]
console.log(newArr); // [1, 2]
```

### Array-Like Objects

Objects that **look like arrays** (have numeric indexes and a `length` property) but **lack array methods**.

Examples:

- `arguments` object inside functions
    
- `NodeList` from `querySelectorAll`
    
- `HTMLCollection`
    

You can convert them to real arrays using:

```js
const realArr = Array.from(arguments);
const arr2 = [...document.querySelectorAll("div")];
```

### Typed Arrays (`Uint8Array`, etc.)

Typed arrays are used for **handling binary data** efficiently — like images, buffers, or WebGL data.

Common types:

- `Int8Array`, `Uint8Array`
    
- `Int16Array`, `Uint16Array`
    
- `Int32Array`, `Uint32Array`
    
- `Float32Array`, `Float64Array`
    

They store **fixed-size binary data** in contiguous memory, unlike normal JS arrays.

```js
const bytes = new Uint8Array(4);
bytes[0] = 255;
console.log(bytes); // Uint8Array(4) [255, 0, 0, 0]
```

Often used with `ArrayBuffer` for low-level data manipulation:

```js
const buffer = new ArrayBuffer(8);
const view = new Int32Array(buffer);
view[0] = 42;
console.log(view[0]); // 42
```
