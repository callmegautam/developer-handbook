
### `if`, `else`, `switch`

Used to control execution paths based on conditions.

```js
let temp = 25;

if (temp > 30) {
  console.log("Hot");
} else if (temp > 20) {
  console.log("Warm");
} else {
  console.log("Cold");
}
```

#### `switch`

Evaluates an expression and matches it against multiple cases.  
`break` prevents fall-through behavior.

```js
let day = 2;
switch (day) {
  case 1: console.log("Mon"); break;
  case 2: console.log("Tue"); break;
  default: console.log("Other");
}
```

### `for`, `while`, `do...while`

#### `for` loop

Runs a block multiple times with a counter.

```js
for (let i = 0; i < 3; i++) {
  console.log(i); // 0, 1, 2
}
```

#### `while` loop

Executes while a condition is true.

```js
let i = 0;
while (i < 3) {
  console.log(i);
  i++;
}
```

#### `do...while` loop

Executes **at least once**, even if the condition is false.

```js
let count = 0;
do {
  console.log(count);
  count++;
} while (count < 3);
```

---

### `break`, `continue`

- `break` → exits the loop entirely.
    
- `continue` → skips the current iteration and continues with the next one.
    

```js
for (let i = 1; i <= 5; i++) {
  if (i === 3) continue; // skips 3
  if (i === 5) break;    // stops at 5
  console.log(i);        // 1, 2, 4
}
```

### `for...of` vs `for...in`

#### `for...of`

Iterates over **values** of iterable objects (arrays, strings, maps, sets).

```js
const arr = [10, 20, 30];
for (let value of arr) console.log(value);
// 10, 20, 30
```

#### `for...in`

Iterates over **keys (property names)** of objects.

```js
const obj = { a: 1, b: 2 };
for (let key in obj) console.log(key, obj[key]);
// a 1
// b 2
```

> Avoid `for...in` on arrays — it iterates over keys as strings, not guaranteed in order.

---

### Labelled Statements

Used to name loops and enable breaking out of nested loops directly.

```js
outer: for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 1) break outer;
    console.log(i, j);
  }
}
```

`break outer;` exits both loops at once.  
Rarely used, but helpful for complex nested control flows.
